# 使用物件儲存空間保存設定與權杖

可以選擇使用 S3 相容的物件儲存來託管設定與身分驗證資料。

**環境變數**

| 變數                     | 是否必填 | 預設值             | 說明                                                                                                                     |
|--------------------------|----------|--------------------|--------------------------------------------------------------------------------------------------------------------------|
| `MANAGEMENT_PASSWORD`    | 是       |                    | 管理面板密碼（啟用遠端管理時必需）。                                                                             |
| `OBJECTSTORE_ENDPOINT`   | 是       |                    | 物件儲存空間的存取端點。可使用 `http://` 或 `https://` 字首指定通訊協定；省略時預設使用 HTTPS。                                      |
| `OBJECTSTORE_BUCKET`     | 是       |                    | 用於存放 `config/config.yaml` 與 `auths/*.json` 的儲存貯體名稱。                                                        |
| `OBJECTSTORE_ACCESS_KEY` | 是       |                    | 物件儲存空間帳號的存取金鑰 ID。                                                                                              |
| `OBJECTSTORE_SECRET_KEY` | 是       |                    | 物件儲存空間帳號的祕密存取金鑰。                                                                                          |
| `OBJECTSTORE_LOCAL_PATH` | 否       | 目前工作目錄 (CWD) | 本機副本根目錄；服務會寫入 `<值>/objectstore`。                                                                         |

**工作流程**

1. **啟動階段：** 解析端點位址與通訊協定字首，建立使用 Path-Style 模式的 MinIO 相容用戶端；若儲存貯體不存在，則自動建立。
2. **本機副本：** 在 `<OBJECTSTORE_LOCAL_PATH 或目前工作目錄>/objectstore` 維護可寫快取，並同步 `config/config.yaml` 與 `auths/`。
3. **初始化：** 若儲存貯體中沒有設定檔，便以 `config.example.yaml` 為範本產生 `config/config.yaml` 並上傳。
4. **雙向同步：** 本機變更會上傳至物件儲存空間，遠端物件也會同步回本機，以確保檔案監聽、管理 API 與 CLI 命令列使用一致的內容。
